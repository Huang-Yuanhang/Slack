Local Environments

Our team develops WordPress websites using the Underscores template in conjunction with Git, GitHub, Visual Studio Code, XAMPP (for local server environments), various databases (for data management), as well as Discord, Trello and Slack for team communication and management.

The development of our product was done by utizing XAMPP v3.3.0 as a open-source server package where we could stimulate a live server environment on a local machine, thus, local environemnt, providing the necessary infrastructure as we installed and deployed the chosen content management syetem (CMS), WordPress 6.4.2, the lateest version at the time, which offers us flexible and user-frendly interface. Although, we did/could not utilize the full capability of WP plug-ins as we were tasked with creating product through HTML, JS, PHP, and CSS alone where the entire codebase was crafted and refined in Visual Studio Code ;streamlining the development process with its code editing, debugging, and GitHub Source Control in real time where our agile team can update our work stimulatiously.

It is worth to mention that the Underscores (_s) theme provided us with a minimalist foundation, allowing for the creation of a unique and tailored theme without the hindrance of superfluous design elements where we utilized this to create our full functioning website.

In summary, for deployment and iterative development, the project was hosted locally via XAMPP integrating with WordPress as Local Environment, embeded in the XAMPP "htdocs" folder. With https://localhost/wordpress-6.4.2/wordpress/ serving as the staging URL, this approach ensured thorough testing and refinement in a controlled environment, allowing for testing and review of all WP components prior to live deployment.
